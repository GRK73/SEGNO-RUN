\# 기획서 : Plan-B RHYTHM

## 1. 게임 개요
* **장르:** 횡스크롤 태그(Tag) 액션 리듬게임
* **플랫폼:** PC 웹 브라우저 (GitHub Pages 배포용 정적 웹사이트)
* **핵심 컨셉:** 상/하 2레인 타격 액션에 '키보드+마우스'라는 비대칭 조작을 도입하고, 마우스 휠을 활용해 보컬 파트와 기믹에 맞춰 캐릭터를 교체하는 신감각 멀티태스킹 리듬게임.

## 2. 핵심 게임플레이 시스템

### 2.1. 2레인 비대칭 런(Run) 타격
* 화면 우측에서 좌측으로 노트가 다가오는 횡스크롤(뮤즈대시 스타일) 방식을 다시 채택합니다.
* 상단(공중)과 하단(지상) 2개의 레인으로 나뉘며, 각 레인의 입력 장치(키보드/마우스)를 완전히 분리하여 좌뇌와 우뇌를 동시에 자극하는 독특한 타격감을 제공합니다.

### 2.2. 유동적 로스터 및 태그(Tag) 시스템
* **곡별 맞춤 로스터:** 곡의 특성과 보컬 파트에 따라 1인(솔로), 3인, 4인 등 참여 캐릭터의 수가 다르게 배정됩니다.
* **타격 조건 (이중 판정):** 다가오는 노트의 색상(주인)과 현재 활성화된 캐릭터가 일치하는 상태에서, 해당 레인에 맞는 장치로 타격해야만 처리됩니다. 불일치 시 Miss 판정이 발생합니다.

### 2.3. 채보 기반 휠 스위칭 (Wheel-driven Switching)
* 캐릭터 전환은 단순한 상태 변경이 아닌, 마우스 휠을 굴려 처리하는 특수 '기믹 노트'로 작용합니다.
* 휠을 위아래로 굴리는 직관적인 조작을 통해 끊김 없이 캐릭터를 순환시키며 액션을 이어나갑니다.

## 3. 조작 체계 (키보드 & 마우스 비대칭 제어)

### 3.1. 타격 조작 (비대칭 분리)
* **하단 레인 (지상 / 키보드):** `F` 또는 `J` 등 편한 키보드 자판 (왼손 권장)
* **상단 레인 (공중 / 마우스):** 마우스 좌클릭 또는 우클릭 (오른손 권장)
* *참고:* 유저의 성향(왼손잡이/오른손잡이)에 따라 키보드와 마우스의 상/하단 역할을 설정 메뉴에서 반전시킬 수 있도록 지원합니다.

### 3.2. 캐릭터 전환 조작 (마우스 휠)
* **우측 순환:** `마우스 휠 위로 굴리기 (Scroll Up)`
* **좌측 순환:** `마우스 휠 아래로 굴리기 (Scroll Down)`
* **가변 순환 로직:** 곡에 배정된 캐릭터 수에 맞춰 순환 배열이 동적으로 바뀝니다.
* *기술적 주의:* 브라우저의 기본 화면 스크롤 동작이 발생하지 않도록 Canvas 위에서 발생하는 `wheel` 이벤트에 대해 `e.preventDefault()` 처리가 반드시 필요합니다.

## 4. UI 및 시각적 피드백

* **비대칭 기어 UI:** 키보드로 치는 라인과 마우스로 치는 라인의 시각적 디자인(아이콘, 이펙트 형태 등)을 다르게 하여 어떤 장치로 쳐야 하는지 뇌가 빠르게 인지하도록 돕습니다.
* **휠 덱(Wheel Deck) 인디케이터:** 화면 하단 또는 캐릭터 주변에 휠의 회전 방향(위/아래)에 맞춰 돌아가는 원형 룰렛 형태의 UI를 배치하여, 스크롤 시 어떤 캐릭터가 나오는지 직관적으로 표시합니다.
* **스위칭 예고선 (Warning UI):** 스위칭 구간이 다가오면 화면에 마우스 휠 아이콘 형태의 힌트나 타겟 캐릭터의 색상 이펙트가 발생하여 조작을 준비하게 합니다.

## 5. 기술 스택 및 배포 환경 (GitHub Pages 최적화)

### 5.1. 프레임워크 및 언어
* **빌드 툴:** `Vite` (빠른 번들링과 GitHub Pages 정적 배포에 최적화)
* **UI 프레임워크:** `React` (로비, 곡 선택, 설정 창 등 DOM 기반 정적 UI 구현용)
* **언어:** `TypeScript` (마우스 휠과 키보드 동시 입력 이벤트 처리, 가변 로스터 상태 관리의 안정성 확보)

### 5.2. 인게임 렌더링 및 오디오 (Core Engine)
* **그래픽 엔진:** `PixiJS` (WebGL 기반)
* **오디오 엔진:** `Web Audio API`

### 5.3. 성능 최적화 (Optimization)
* **이벤트 리스너 최적화:** 한 번의 휠 '틱(Tick)'을 정확하게 1회의 캐릭터 전환으로 인식하도록 디바운싱/쓰로틀링 제어 로직을 적용합니다.
* **오브젝트 풀링 (Object Pooling):** 노트 및 타격 이펙트를 풀(Pool)에서 꺼내어 사용함으로써 가비지 컬렉션(GC) 렉을 방지합니다.
* **메인 루프:** `requestAnimationFrame`을 통해 브라우저 주사율에 동기화된 부드러운 횡스크롤을 보장합니다.

## 6. 채보 데이터 구조 (JSON Schema)

이 게임의 시스템을 완벽히 소화하기 위해, 곡의 기본 정보(메타데이터)와 각 노트의 속성(레인, 조작 장치, 담당 캐릭터, 스위칭 기믹)을 분리하여 설계한 JSON 규격입니다.

```json
{
  "meta": {
    "title": "Song Title",
    "artist": "Artist Name",
    "bpm": 128,
    "offset": 0,             // 첫 박자 시작 지점 (ms)
    "roster": [0, 1, 2]      // 이 곡에 참여하는 캐릭터 ID 배열 (예: 3인 체제)
  },
  "notes": [
    {
      "time": 1200,          // 판정선에 닿아야 하는 오디오 시간 (ms)
      "lane": 0,             // 0: 하단(키보드), 1: 상단(마우스)
      "type": "normal",      // 일반 타격 노트
      "characterId": 0       // 0번 캐릭터 상태여야 타격 가능
    },
    {
      "time": 1800,
      "lane": 1,
      "type": "normal",
      "characterId": 0
    },
    {
      "time": 2400,
      "lane": "any",         // 레인 무관 (어디서든 스위칭 가능하도록 처리)
      "type": "switch",      // 마우스 휠을 굴려야 하는 '스위칭 기믹 노트'
      "targetCharId": 1      // 1번 캐릭터로 전환 성공 시 판정 인정
    },
    {
      "time": 2800,
      "lane": 0,
      "type": "normal",
      "characterId": 1       // 전환된 1번 캐릭터의 노트 시작
    }
  ]
}
